## morning_briefing 关键修复（本轮重点）

### 问题1：subprocess curl 返回 HTTP 501
- 原因：subprocess.run(cmd) 对 MCP 82.156.17.205 返回空，urllib 直连正常
- Accept header 必须为 "application/json, text/event-stream"（否则 HTTP 400）
- 解决：urllib 替代 subprocess，不指定 method 参数

### 问题2：SSE JSON 提取失败
- 响应格式：`event: message\r\ndata: {"jsonrpc":"2.0",...}`
- 原正则 `r'data:\s*({.+?})\s*$'` 匹配失败
- 解决：`raw.find('data:')` → `json.loads(raw[json_start+5:].strip())`

### 问题3：价格解析错误
- brief 返回多个 "当日:" 行（价格、涨幅、振幅等）
- 价格为 `- 当日: 18.350 最高: 18.500 最低: 17.000`
- 原代码匹配第一个含 "当日:" 的行（含非价格行如 "6.91"、"1.25"）
- 解决：价格行必须含 "最高:" 关键字；`float(ln.split(":",1)[1].strip().split()[0])` 取数字部分

## 18:15-18:20 重大发现：datetime.now() 是 Python bug

### 问题
- `datetime.now()` 在 Python 3.10 标准库中不存在
- 标准库只有 `datetime.datetime.now()`
- `import datetime` 后，`datetime.now` 不存在
- 症状：`AttributeError: module 'datetime' has no attribute 'now'`

### 受影响文件（13个）
- stop_loss_monitor.py, market_open_scan.py, market_close_scan.py
- morning_briefing.py, portfolio_monitor.py, stock_picker.py
- stock_pool_updater.py, activity_logger.py, cron_health_check.py
- process_inbox.py, portfolio_history.py, portal_task_hook.py, stock_scan.py

### 修复
- 批量替换 `datetime.now()` → `datetime.datetime.now()`
- 结果：所有脚本 dry-run 测试通过 ✅

### 根因推测
某个旧版 Python 或 datetime 模块补丁允许 `datetime.now()` 作为 `datetime.datetime.now()` 的别名，但 Python 3.10 标准库不支持。

### 新发现
stock_scan.py 有双重嵌套 bug：`datetime.datetime.datetime.now()` - 说明脚本被多次编辑导致嵌套。

### 新规则
所有新脚本必须用 `from datetime import datetime` + `datetime.now()`，或 `import datetime` + `datetime.datetime.now()`。禁止混用。
