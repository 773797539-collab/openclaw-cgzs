## 03:00-03:10 自驱工作

### write工具破坏\n教训（重要）
- write工具把Python字符串中的\n转义成\\n（两个字符），导致f.write("\\n")变成字面"\\n"
- 解决：用python3 heredoc或python3 -c写入文件，避免用write工具写代码文件
- 精准修复：sed -i '54s/.*/        f.write("\\n")/' + python3 -m py_compile验证

### .gitignore彻底解决方案
- 运行产物(portal/status/*.json, internal_pool.json, reports/*.log)必须不追踪
- .gitignore写规则后，还需git rm --cached <file>移除已追踪记录
- git check-ignore -v <file>验证是否生效
