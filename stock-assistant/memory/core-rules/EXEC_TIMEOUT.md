# 执行超时与重试策略 - stock-assistant

**层级**: 第二层 - 流程与模板
**创建**: 2026-03-26 08:33 GMT+8

---

## 一、执行超时配置

| 操作类型 | 超时时间 | 说明 |
|----------|----------|------|
| akshare 数据获取 | 30秒 | 单只股票/指数查询 |
| 批量市场扫描 | 120秒 | 全量涨跌停/异动扫描 |
| 脚本执行（一般） | 60秒 | 一般Python/shell脚本 |
| 文档写入 | 10秒 | 本地文件IO |
| 外部API调用 | 30秒 | 非akshare的HTTP请求 |

---

## 二、重试策略

| 场景 | 重试次数 | 间隔 | 说明 |
|------|----------|------|------|
| 数据源连接失败 | 3次 | 5秒 | 网络抖动 |
| akshare 接口超时 | 2次 | 10秒 | 服务器端问题 |
| 数据为空 | 1次 | 3秒 | 可能是查询条件问题 |
| 非交易时段 | 0次 | - | 直接跳过，不报错 |

---

## 三、非交易时段处理

- akshare 在非交易时段返回错误属正常现象
- 09:00 前和 16:00 后的行情查询：捕获异常，结果标记"非交易时段数据"
- 不发送 ERROR 通知，只在报告中标注

---

## 四、超时处理流程

```
执行 → 超时 → 重试N次 → 仍失败 → 标记结果 → 记录日志 → 通知用户（严重时）
```
