# 核心规则层 - stock-assistant

**层级**: 第一层 - 核心规则
**说明**: 项目目标、边界、口径、禁区、验收硬规则

---

## 一、项目边界

- **业务范围**: A 股辅助决策支持
- **禁止**: 自动下单、直连券商账户
- **输出**: 分析、摘要、提醒、报告，不做交易执行

---

## 二、口径规则

- **默认输出语言**: 中文
- **结论优先**: 先给结论，再给详情
- **数据来源**: 优先使用可靠的财经数据 API
- **数据延迟**: 接受 15 分钟延迟，不做实时交易

---

## 三、禁区

1. 不执行任何形式的下单指令
2. 不存储完整资金账户信息
3. 不在无日志情况下执行高风险操作
4. 不向未授权第三方透露用户数据

---

## 四、验收硬规则

1. 任何任务执行前必须有 trace 记录
2. 任何外部 API 调用必须记录请求和响应摘要
3. 失败必须写失败样本到 failure-cases
4. 通知必须留存记录
