## 一、当前主 agent 系统提示词策略

### stock-main（前台总控）
- 职责：理解用户需求、分派任务、汇总结果
- 优先读权限、调度权限
- 决策链：接收 → 理解 → 分派 → 汇总 → 通知

### stock-research（研究）
- 职责：资料检索、规则整理、方案草拟
- 优先网络/读取/总结权限
- 输出：结构化信息摘要

### stock-exec（执行）
- 职责：脚本执行、文件改动、数据产出
- 需配套留痕和回滚
- 必须先验证再交付

### stock-review（验收）
- 职责：检查结果是否达到预期
- 只读优先，避免边验边改

### stock-learn（学习复盘）
- 职责：复盘失败样本、沉淀经验
- 读取项目结果、日志、失败样本

## 三、优化方向

1. **减少幻觉**：要求 stock-review 必须有数据依据
2. **减少废话**：结论优先原则写入 system prompt
3. **留痕强化**：执行结果必须有 trace 记录
4. **通知精简**：禁止重复通知，同类合并
