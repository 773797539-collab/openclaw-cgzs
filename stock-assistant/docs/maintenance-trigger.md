# 主文档自动维护触发规则

**文件**: maintenance-trigger.md
**创建**: 2026-03-26 04:05 GMT+8
**依据**: 实施总文档 v2.2 第九章

---

## 一、触发条件

每次以下事件发生时，必须自动触发主文档更新检查：

| 事件 | 触发动作 |
|------|----------|
| 新增/删除文件 | 检查是否影响目录结构，更新 10.1/10.2 |
| 新增/删除 agent | 更新 agent-config.md + 主文档第六章 |
| 新增/修改 cron | 更新 05_OpenClaw更新与补丁日志.md |
| 发现失败样本 | 写入 03_踩坑记录与失败样本.md |
| 任务池重大变更 | 更新 project-overview.md 当前阶段 |
| 配置文件变更 | 记录到 02_已验证有效命令与流程.md |

---

## 二、更新流程

```
事件发生 
  → 识别事件类型 
  → 确定目标文件 
  → 执行更新 
  → 追加变更日志 
  → 推送通知（如需要）
```

---

## 三、强制同步规则

以下情况必须在完成主文档更新前，停止所有后续动作：

- 新增 agent
- 修改目录结构
- 修改核心规则（core-rules）
- 启停插件

---

## 四、更新检查清单

每次主文档更新后，检查：
- [ ] Markdown 主文档已同步
- [ ] 变更日志已追加
- [ ] 相关落地文件已同步更新
- [ ] 需要用户确认的事项已通知

---

- `docs/changelog/project-change-log.md`
- `stock-assistant/docs/01-05系列文件`
