**状态**: ✅ 已解决（禁用插件）

wecom-openclaw-plugin 在启动时加载失败。

- 后续引入插件前应先评估兼容性

### 样本 #003 - akshare 直连网络不稳（服务器访问外网受限）

**时间**: 2026-03-27 13:30 GMT+8
**严重程度**: 中（影响数据获取）
**状态**: ✅ 已解决

#### 问题描述
akshare v1.18.38 直连东方财富等数据源时，网络请求超时或 RemoteDisconnected。

#### 原因分析
云服务器访问外网受限，akshare 直连方案不可用。

#### 解决方式

- 云服务器可能无法直接访问外网数据源
- MCP/封装 API 优于直连
- akshare 适合有外网权限的环境

### 样本 #004 - curl Accept header 多行参数覆盖 bug

**时间**: 2026-03-27 15:30 GMT+8
**严重程度**: 低（调试层面）
**状态**: ✅ 已解决

#### 问题描述
MCP HTTP 接口调用一直返回 406 Not Acceptable，但单独测试 curl 命令正常。

#### 原因分析
subprocess 调用 curl 时，用两个独立的 `-H Accept:` 传参，第二个会覆盖第一个：
```python
["-H", "Accept: application/json",
 "-H", "Accept: text/event-stream"]  # 覆盖了前者
```

#### 解决方式
合并为一个 header：
```python
["-H", "Accept: application/json, text/event-stream"]
```

#### 教训沉淀
- curl 传参注意 Accept 头：必须同时包含 `application/json` 和 `text/event-stream`
- 两次 `-H` 不会合并，后者会覆盖前者
