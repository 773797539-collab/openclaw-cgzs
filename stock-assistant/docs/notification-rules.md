# 通知分类与降噪规则

## 通知分类

### 🔴 必须推送（高优先级）
1. **真实失败** - 脚本/任务执行报错（非重试可解决的）
2. **真实阻塞** - 依赖项缺失、网络不通、凭证失效
3. **待审批** - 需要人工授权的操作

### 🟡 选择推送（中优先级）
4. **结果摘要** - 批量任务完成后的汇总
5. **持仓警告** - 止损线触发、异常亏损

### 🟢 仅进门户（低优先级）
6. **心跳** - 定时健康检查正常
7. **过程信息** - 脚本执行进度、中间结果
8. **研究结论** - 量化分析结果（进门户报告）
9. **定时任务触发** - cron 任务启动

## 抑制规则

| 场景 | 抑制条件 | 动作 |
|------|---------|------|
| HEARTBEAT 正常 | 距上次推送<30min | 静默 |
| 批量扫描 | 单只股票超时 | 不单独通知，汇总后通知 |
| 重复失败 | 同一任务3次内失败 | 第1次通知，第2-3次静默等待重试 |
| GitHub push | 网络超时<60s | 自动重试3次，失败才通知 |
| MCP 超时 | 单次<30s | 静默（重试即可）|
| GitHub sync | 120s内完成 | 静默 |

## 通知格式

必须包含：
- 触发原因（明确）
- 影响范围（具体）
- 建议动作（明确）

禁止：
- "执行中""处理中"等无意义状态
- 重复刷屏的过程信息
- 无证据的推测性结论
