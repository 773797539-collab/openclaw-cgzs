## 问题描述

止损告警日志显示同一告警（立达信-16.4%）被重复触发多次（10条历史，时间间隔~1.5-2小时）：

```
2026-03-27 00:10 : 🔴 止损触发！立达信(605365) 现价¥17.26（-16.4%）
2026-03-26 22:10 : 🔴 止损触发！立达信(605365) 现价¥17.26（-16.4%）
2026-03-26 20:40 : 🔴 止损触发！立达信(605365) 现价¥17.26（-16.4%）
```

## 初步分析

check_and_alert() 有去重逻辑（`log[last_key] != val`），但系统重启后 log 被重置，`_last_alert` 键丢失，导致重复告警。

## 根因

stop_loss_monitor.py 的日志保存逻辑：
- 每次运行读入旧 log
- 提取 `_code_last_alert` 键
- **但保存时用 `json.dump(log, f)` 全量覆盖**，且 log 初始只有 `last_check/alerts/history` 三个键
- 如果 `_last_alert` 键在旧 log 中但因为某些原因没有正确传递过来，就会重复告警

## 计划修复方案

在 check_and_alert() 开头加载旧 `_last_alert` 到独立字典，每次告警比对后立即写回，不要等最终保存。

30分钟，修复+测试
