## 二、任务池目录

```
tasks/
├── todo/          # 待办
├── doing/         # 进行中
├── done/          # 已完成
├── blocked/       # 阻塞
└── approval/      # 待审批
```

---

## 三、任务文件格式

每个任务一个文件，放在对应状态目录：

**文件命名**: `{timestamp}-{task-type}-{short-description}.md`

**任务文件内容**:
```markdown
# 任务标题

- **任务ID**: 
- **类型**: 
- **优先级**: 
- **创建时间**: 
- **状态**: 
- **阻塞原因**: (如有)
- **执行记录**: (完成后填)
- **结果**: (完成后填)
```

---

## 四、流转规则

1. 新任务 → `todo/`
2. 开始执行 → `doing/`（同时记录开始时间）
3. 完成 → `done/`（记录结果和耗时）
4. 阻塞 → `blocked/`（写清阻塞原因和所需输入）
5. 需审批 → `approval/`（写清风险和影响）

## 六、持续作业模式（2026-03-28 16:49 强制启用）

**来源**：用户指令，不得以任何理由停机。

### 核心规则
- 只要 token remaining > 0，就继续工作，不停止
- 禁止出现：待机/空闲/等待/暂停/本轮结束
- 完成任务 → 立即找下一个 → 循环不停

### 找任务顺序（每次完成后按此顺序）
1. `tasks/todo/` 待办队列
2. `tasks/blocked/` 阻塞队列（检查是否有可恢复项）
3. `tasks/approval/` 审批队列（检查是否无需人工即可推进）
4. 都没有 → 自动生成成长型任务

### 成长型任务优先级（无外部任务时）
1. 系统自查与异常修复
2. 状态页/通知链路/自动播报可靠性优化
3. 任务池去重与恢复机制治理
4. 记忆整理、总结沉淀、文档补全
5. skills/plugins/workflow 研究与可落地评估
6. 提示词/执行链路/验收机制优化
7. 生成下一批可执行任务并回填任务池

### 停止条件（只有这4种）
1. token 真正耗尽（remaining = 0）且已确认
2. 用户明确发出停止指令
3. 外部依赖全部失败 + 降级方案仍无法继续
4. 系统故障需人工介入

### 阻塞处理
- 单问题卡住 > 3分钟 → 立即切换并行子任务
- 需用户确认 → 放入审批池，但主流程不停
- 夜间/不在线 → 默认安全保守可回滚，不因等回复而停工
